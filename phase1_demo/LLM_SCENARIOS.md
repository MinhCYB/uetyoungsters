# Student Companion LLM V1 Scenarios

The content layer is tested without network calls or model credentials. All
provider responses are scripted or generated by deterministic templates.

| ID | Condition | Expected behavior | Test coverage |
|---|---|---|---|
| L01 | Valid plan expansion | Preserve task ID, skill/career and exact minutes | `test_template_plan_step_minutes_equal_engine_time` |
| L02 | Malformed JSON twice | One retry, then validated template fallback | `test_malformed_json_retries_then_falls_back` |
| L03 | Provider changes duration | Reject output and preserve engine duration | `test_changed_engine_plan_decision_uses_fallback` |
| L04 | Provider changes skill | Reject output and preserve engine skill | `test_changed_engine_plan_decision_uses_fallback` |
| L05 | Provider timeout | One retry, no raw exception, template fallback | `test_provider_failure_retries_then_falls_back` |
| L06 | Provider exception | Same safe fallback path as timeout | `test_provider_failure_retries_then_falls_back` |
| L07 | Duplicate/prior question fingerprint | Reject generated package and create non-repeating fallback | `test_duplicate_generated_fingerprint_uses_fallback`, `test_prior_question_fingerprint_is_not_reused` |
| L08 | Feedback depth is hint | Give a hint without worked solution steps | `test_hint_depth_has_no_full_worked_solution` |
| L09 | Feedback depth is worked solution | Require explicit, student-friendly steps | `test_worked_solution_depth_has_explicit_steps` |
| L10 | Template mode or failed provider | Mark content as `template_fallback`, never as live AI | plan/reassessment/feedback fallback tests |
| L11 | Prompt contains private IDs in optional text | Redact teacher/tenant/evidence identifiers; omit full profile | `test_llm_prompt_privacy.py` |
| L12 | Existing provider unavailable | Fail closed and run deterministic fallback without crashing | `test_existing_provider_unavailable_falls_back_without_crash` |

Additional scenarios cover schema errors, score/time invariants, question type
rules, difficulty bounds, judgmental/career/medical language, score promises,
content IDs, CLI output, full-plan expansion, and 20-request determinism.
